# Финальные правки

Логика готова: все 10 пунктов ТЗ работают, persist на месте. Осталась полировка. Она разбита на **две отдельные задачи** (как и весь проект — одна задача = одна ветка = один PR):

1. **Чистка кода** — README, нейминг, мелочи, форматтер.
2. **UI** — привести интерфейс к нормальному виду (ориентир — приложенная картинка).

**Порядок:** сначала чистка, потом UI. В чистке настраивается prettier — после этого вся UI-работа форматируется в едином стиле автоматически. Если сделать UI первым, prettier потом перелопатит все стили и дифф будет грязный. Не сваливай обе задачи в один PR.

> Про UI: **структуру компонентов не трогаешь**. `App / TaskForm / TaskFilters / TaskList / TaskItem / TaskStats` остаются как есть. Вторая задача — только стили (SCSS) и мелкие правки разметки внутри уже существующих компонентов. Переписывать ничего не нужно.

## Git (одинаково для обеих задач)

```bash
git checkout dev
git pull
git checkout -b <имя-ветки>      # ветку всегда от свежего dev
# ...работа, осмысленные коммиты по подпунктам...
git push -u origin <имя-ветки>
```

Дальше PR `<ветка>` → `dev`. Перед каждым push прогоняй `npm run build` и `npm run lint` — оба должны быть зелёными.

## Задача 1 — ветка `refactor/cleanup`

Делается первой. Маленькая, закрывает хвосты и DoD.

**1. README (обязательно — пункт Definition of Done)**
Сейчас в README дефолтный текст Vite-шаблона. Замени на свой:
- 1–2 строки, что это за проект;
- команды запуска: `npm install`, затем `npm run dev`;
- по желанию `npm run build`, `npm run lint`.

**2. Нейминг**
Имена должны говорить, что делает функция/проп.
- `getTask` → `addTask` (функция добавляет задачу, а не получает);
- `ToggleData` → `toggleTask` (с большой буквы читается как компонент);
- проп `cb` в `TaskForm` → `onAddTask`;
- колбэки-пропсы по смыслу: `onToggle`, `onDelete`.

**3. Проверка пустой задачи**
Сейчас `text.replaceAll(" ", "")` — ловит только пробелы (не табы/переносы) и сохраняет текст с пробелами по краям. Используй `text.trim()`: и в проверке, и при сохранении (`text: text.trim()`).

**4. Обновление состояния без мутаций**
В `toggle` и `delete` уже правильно (`setData(prev => ...)`). В `addTask` всё ещё `setData([...data, ...])` — приведи к функциональному апдейтеру `setData(prev => [...prev, ...])`. Заодно загляни, почему массив обновляют через новый массив, а не `data.push(...)` — это базовая идея React.

**5. Мелочи**
- `useEffect(() => localStorage.setItem(...))` со стрелкой без тела выглядит как cleanup-функция. Поставь тело: `useEffect(() => { localStorage.setItem(...) }, [data])`.
- ключ localStorage `"Data"` → осмысленный нижний регистр, например `"tasks"`.

**6. Prettier + ESLint (хвост с первой ветки)**
`prettier` и `eslint-config-prettier` стоят в зависимостях, но не настроены:
- создай `.prettierrc.json`:
```json
{ "semi": false, "singleQuote": true, "printWidth": 100, "trailingComma": "all" }
```
- подключи `eslint-config-prettier` **последним** в `eslint.config.js`;
- один раз прогони `npx prettier --write .`.

**Примерные коммиты:**
`docs: README с командами запуска` · `refactor: понятный нейминг функций и пропсов` · `fix: trim вместо replaceAll` · `refactor: функциональный апдейтер в addTask` · `chore: настройка prettier + eslint-config-prettier`

## Задача 2 — ветка `feature/ui`

Делается после мёржа первой (ветку от обновлённого `dev`). Ориентир — приложенная картинка `ui-target.png`. Пиксель-в-пиксель не нужно: важно повторить сетку, отступы и состояния. Только стили, структура прежняя.

**1. Контейнер — главный фикс.** Сейчас всё растянуто на всю ширину экрана. Оберни приложение в центрирующий контейнер: `max-width: ~560px`, `margin: 0 auto`, вертикальный стек с равными отступами между секциями (`gap`), общий отступ страницы по краям. Одно это убирает 80% «сломанности».

**2. Форма.** Сейчас гигантская коричневая плашка с крошечным инпутом. Инпут и кнопка — **в одну строку**: инпут тянется (`flex: 1`), кнопка по содержимому, одинаковая высота (~46px), скругления, нормальные паддинги. Никакой плашки-обёртки. Добавь `placeholder` и видимый `:focus`.

**3. Фильтры.** Сейчас не видно, какой активен. Сделай сегментированный переключатель (три кнопки в одной плашке) и **подсвечивай активный** (фон + цвет). `mode` уже в `App` — вешай класс `active`, когда значение совпадает.

**4. Статистика.** Сейчас она в самом низу, мелкая, со стрелочками — похоже, числа выводятся в `input type="number"`. Убери инпуты: это просто **текст** (`Всего: 3 · Выполнено: 1 · Осталось: 2`). Размести **над списком**. Кнопку «Удалить выполненные» сделай скромной (текстовая, приглушённо-красная), а не во всю ширину.

**5. Карточка задачи.** Компактная строка: `[чекбокс] [текст ——— тянется] [Удалить]`, по центру по вертикали, отступы через `gap`. Чекбокс ~20px с `accent-color`. **Выполненная задача — зачёркнута и приглушена** (`line-through` + серый). Сейчас визуально не видно, что задача выполнена, — а это пункт ТЗ. Кнопку удаления сделай неброской (серый текст → красный при наведении), без двухстрочной красной плашки.

**6. Пустой список** (из «опционально» ТЗ). Когда задач нет или под фильтром ничего не попало — покажи короткий текст «Пока нет задач» / «Ничего не найдено» вместо пустоты.

**7. Цвета/шрифт.** В картинке — облагороженная версия твоей тёплой гаммы: светлый фон, белые карточки, **один** акцентный цвет на кнопки и активный фильтр. Главное правило: один акцент, а не пять оттенков коричневого.

**Примерные коммиты:**
`feat(ui): центрирующий контейнер и сетка` · `feat(ui): форма в одну строку` · `feat(ui): активное состояние фильтров` · `feat(ui): статистика над списком` · `feat(ui): карточки задач + зачёркивание выполненных` · `feat(ui): пустое состояние списка`
